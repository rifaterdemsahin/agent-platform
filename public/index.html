Agent Lifecycle & Architecture

1. Spec
Define your agent in agents/<name>/agent.json — name, version, description, skills, runtime, triggers.
2. Code
Implement skills in agents/<name>/index.js. Each skill is an async function that receives (params, context).
3. Runtime
runtime/agent-loader.js scans agents/ at startup. Agents are cached in memory and ready to invoke.
4. API
Express serves /api/agents/:name/invoke. Agents receive the Anthropic LLM client via context.
5. Background
Agents declare "background": true with pollingIntervalMs. External cron or scheduler pings invoke endpoints.
6. Deploy
Push to main → GitHub Actions runs flyctl deploy → Fly.io publishes HTTPS.

Updating Agent Skills

Skills live in agents/<name>/skills/<skill>.js. Edit the skill file, add it to the agent's skills array in agent.json, push to GitHub, and Fly.io redeploys automatically.
